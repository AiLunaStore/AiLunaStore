# MedSurg Checklist Brain Sheet Specifications

## Document Purpose
Detailed specifications for each section of the MedSurg checklist brain sheet, including checklist options, layout, color coding, and human outline diagram.

## Overall Sheet Specifications

- **Page size**: 8.5" × 11" portrait
- **Margins**: 0.25" on all sides
- **Orientation**: Portrait
- **Font**: Segoe UI, 8pt base
- **Color coding**: Per section (see below)
- **Layout**: Two-column (70% left for checklists, 30% right for human outline & notes)
- **Checkbox style**: Square boxes [ ] for multi-select, circles ○ for single-select (pain scale)

## Section 9: Human Outline Diagram (Neutral)

### Purpose
Visual mapping of IV sites, Foley, wounds, drains, etc.

### Design Specifications
- **Silhouette style**: Simple line drawing, front view, no internal anatomy
- **Landmarks**: Dots at shoulders, elbows, wrists, hips, knees, ankles
- **Left/Right labels**: "L" and "R" near shoulders and hips
- **Symbol key** (drawn adjacent to silhouette):
  - ● IV site (with L/R notation)
  - ▲ Foley catheter
  - ✕ Wound/dressing
  - ○ Drain (JP, Hemovac, chest tube)
  - ■ Pressure injury
- **Space around**: Ample whitespace for drawing symbols and notes

### Layout
- Occupies top 60% of right column
- Key placed below silhouette
- Blank lines for additional descriptions

### Color Scheme
- **Outline color**: #34495e (dark blue-gray)
- **Background**: white
- **Border**: #34495e thin solid

## Print Considerations

### Page Breaks
- Entire sheet fits one page; avoid breaking sections.
- Use `break-inside: avoid` in CSS for each section.

### Ink Usage
- Light backgrounds only; avoid solid fills.
- Grayscale printing friendly (colors convert to distinct gray patterns).

### Margins
- 0.25" minimum; keep critical content inside.

## Digital Adaptation Notes

### Tablet Interface
- Touch‑friendly checkboxes (larger tap targets)
- Dropdowns for common selections (e.g., fluid type, rate)
- Auto‑calculate I&O balance
- Voice‑to‑text for notes

### Smart Features
- Auto‑populate from EMR (allergies, code status)
- Alerts for missing assessments (e.g., fall risk unchecked)
- Export to PDF for printing

## Customizability Guidelines

### Base Template (Non‑negotiable)
- All core checklist sections
- Human outline diagram
- Color coding system

### Optional Sections
- **Pre‑op checklist**: NPO after midnight, consent signed, etc.
- **Post‑op checklist**: Incision check, PCA pump, etc.
- **Discharge planning**: Teaching done, referrals, etc.

### Unit‑Specific Variations
- **Ortho**: Add neurovascular checks, TED hose, SCDs
- **Cardiac**: Add telemetry monitoring, edema assessment
- **Oncology**: Add chemo side effects, neutropenic precautions

## Validation Criteria

### Usability Testing
- Nurses can complete sheet for a typical MedSurg patient in < 5 minutes
- All common scenarios covered by checklists (80% rule)
- Human outline intuitive for marking locations

### Efficiency Metrics
- Reduction in free‑text writing (measure character count)
- Fewer missed assessments (self‑reported)
- Improved handoff communication (survey)

### Safety Metrics
- Error rate in documentation (compared to EMR)
- Early detection of changes (e.g., IV infiltration, pain escalation)
