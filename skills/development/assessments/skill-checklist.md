# Skill Proficiency Checklist

## Purpose
Detailed checklist for evaluating proficiency in specific skills.

## How to Use

1. **Select a skill** to evaluate
2. **Work through the checklist** honestly
3. **Count checked items** for each level
4. **Determine proficiency** based on criteria
5. **Identify next steps** for development

## Programming (Python)

### Novice (0-25% complete)
- [ ] Write basic scripts
- [ ] Understand syntax
- [ ] Use variables and types
- [ ] Control flow (if/else, loops)
- [ ] Basic functions

### Competent (25-50% complete)
- [ ] Write modular code
- [ ] Use standard library
- [ ] Handle errors (try/except)
- [ ] Work with files
- [ ] Basic OOP
- [ ] Use third-party packages
- [ ] Debug simple issues

### Proficient (50-75% complete)
- [ ] Design classes effectively
- [ ] Use decorators
- [ ] Context managers
- [ ] List/dict comprehensions
- [ ] Generators
- [ ] Unit testing
- [ ] Virtual environments
- [ ] Package creation
- [ ] Code review
- [ ] Performance profiling

### Expert (75-90% complete)
- [ ] Metaclasses
- [ ] Async/await
- [ ] C extensions
- [ ] Memory management
- [ ] Design patterns
- [ ] Architecture design
- [ ] Optimization techniques
- [ ] Mentoring others
- [ ] Complex debugging

### Master (90-100% complete)
- [ ] Language internals
- [ ] Create frameworks
- [ ] Contribute to CPython
- [ ] Novel patterns
- [ ] Industry leadership
- [ ] Advanced optimizations

**Proficiency Determination:**
- Novice: 0-5 items
- Competent: 6-12 items
- Proficient: 13-22 items
- Expert: 23-31 items
- Master: 32-36 items

## Using This Checklist

### For Self-Assessment
1. Choose skill to assess
2. Check all items you can confidently do
3. Count and determine level
4. Identify unchecked items for development

### For Goal Setting
1. Determine target level
2. Identify all items needed for that level
3. Create development plan
4. Track progress

### For Progress Tracking
1. Re-assess monthly
2. Count new items checked
3. Update proficiency level
4. Adjust development plan
