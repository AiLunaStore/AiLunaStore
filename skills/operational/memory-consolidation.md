# Memory Consolidation

## Skill Category
Operational

## Purpose
Systematically transfer information from episodic memory (daily files) to semantic memory (knowledge) and procedural memory (skills).

## Prerequisites
- Understanding of three-layer memory system
- Access to memory files and knowledge/skills directories
- Basic file editing capabilities

## Procedure

### Weekly Consolidation Process

#### Step 1: Preparation
1. Set aside dedicated time (Sunday works well)
2. Gather recent memory files (past 7 days)
3. Open knowledge and skills directories for editing

#### Step 2: Review Memory Files
1. Read each daily file from the past week
2. Highlight or note significant content:
   - New concepts or insights
   - Important conversations or events
   - Problems solved and solutions
   - Skills practiced or learned
   - Decisions made and rationale

#### Step 3: Extract Semantic Knowledge
For each significant insight:

1. **Concepts:**
   - Create/update file in `knowledge/concepts/`
   - Include definition, description, examples
   - Link to related concepts

2. **People:**
   - Update `knowledge/people/[name].md`
   - Add new information, context, interactions
   - Note relationship changes

3. **Projects:**
   - Update `knowledge/projects/[project].md`
   - Document progress, challenges, decisions
   - Update timeline and next steps

4. **Systems:**
   - Update `knowledge/systems/[system].md`
   - Document configurations, integrations
   - Note issues and solutions

5. **World Knowledge:**
   - Add to `knowledge/world/[topic].md`
   - Include facts, context, sources

#### Step 4: Extract Procedural Knowledge
For each skill or process:

1. **Technical Skills:**
   - Document in `skills/technical/[skill].md`
   - Include step-by-step procedure
   - Add examples and variations

2. **Creative Processes:**
   - Document in `skills/creative/[process].md`
   - Describe workflow, techniques, tools

3. **Analytical Methods:**
   - Document in `skills/analytical/[method].md`
   - Explain approach, tools, interpretation

4. **Social Patterns:**
   - Document in `skills/social/[pattern].md`
   - Describe communication, interaction styles

5. **Operational Routines:**
   - Document in `skills/operational/[routine].md`
   - Outline steps, timing, tools

#### Step 5: Update Long-term Memory
1. Review `MEMORY.md`
2. Add significant events, lessons, growth
3. Update personal insights and reflections
4. Ensure consistency with other memory layers

#### Step 6: Archive and Cleanup
1. Move files older than 30 days to `memory/archive/`
2. Remove redundant or trivial entries
3. Update cross-references between files
4. Verify directory structure integrity

### Monthly Review Process
1. Review all knowledge and skills directories
2. Prune outdated information
3. Consolidate related concepts
4. Update index files and navigation
5. Backup the memory system

## Examples

### Example 1: Consolidating a Learning Experience
**Episodic memory entry:**
"2026-03-15: Learned about three-layer memory system today. It has episodic, semantic, and procedural layers."

**Consolidation:**
1. Create `knowledge/concepts/three-layer-memory.md`
2. Add concept definition, description, benefits
3. Link to related concepts (cognitive architecture, etc.)

### Example 2: Consolidating a Problem Solution
**Episodic memory entry:**
"2026-03-16: Fixed the PDF generation issue by updating the template engine configuration."

**Consolidation:**
1. Update `knowledge/systems/pdf-generation.md`
2. Document the issue and solution
3. Create/update `skills/technical/pdf-generation-troubleshooting.md`

### Example 3: Consolidating a Skill Development
**Episodic memory entry:**
"2026-03-17: Practiced using the new code review workflow. Found it more efficient than the old approach."

**Consolidation:**
1. Create `skills/operational/code-review-workflow.md`
2. Document the new workflow steps
3. Compare with old approach and note improvements

## Variations

### Quick Consolidation (15 minutes)
- Skim daily files for most significant items
- Focus on high-impact knowledge and skills
- Defer detailed work to next full consolidation

### Deep Consolidation (2+ hours)
- Read all daily files thoroughly
- Extract all potentially valuable content
- Update all related knowledge and skill files
- Review and prune existing directories

### Collaborative Consolidation
- Review memory files with team member
- Discuss what's worth preserving
- Get feedback on knowledge organization
- Align on skill documentation standards

## Troubleshooting

### Problem: Too much content to consolidate
**Solution:**
- Prioritize by significance and relevance
- Use quick consolidation for less critical items
- Schedule more frequent, shorter sessions

### Problem: Uncertain where to file something
**Solution:**
- Use best judgment and file where it seems most relevant
- Add cross-references to other potential locations
- Review during monthly consolidation and adjust if needed

### Problem: Inconsistent documentation style
**Solution:**
- Create templates for different file types
- Review and standardize during monthly review
- Update older files gradually over time

### Problem: Falling behind on consolidation
**Solution:**
- Schedule regular time (e.g., Sunday afternoon)
- Set reminders or calendar events
- Start with most recent and work backward
- Accept partial progress rather than perfection
