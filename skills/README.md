# Skills Directory - Procedural Memory Layer

## Purpose
This directory contains your **procedural memory** - skills, workflows, processes, and how-to knowledge that represents your capabilities.

## Structure

### skills/technical/
- Programming languages and frameworks
- System administration
- DevOps and infrastructure
- Technical workflows

### skills/creative/
- Writing and communication
- Design and visualization
- Storytelling and narrative
- Creative processes

### skills/analytical/
- Data analysis
- Research methods
- Problem-solving frameworks
- Decision-making processes

### skills/social/
- Communication patterns
- Relationship management
- Collaboration workflows
- Social intelligence

### skills/operational/
- Daily routines
- Task management
- Productivity systems
- Workflow optimization

## Skill Documentation Format

Each skill should be documented with:

1. **Skill Name**: Clear, descriptive title
2. **Purpose**: What this skill enables
3. **Prerequisites**: Required knowledge or tools
4. **Procedure**: Step-by-step instructions
5. **Examples**: Concrete use cases
6. **Variations**: Different approaches or contexts
7. **Troubleshooting**: Common issues and solutions
8. **Related Skills**: Connections to other capabilities

## Maintenance
- Update when procedures change
- Add new skills as they're developed
- Remove deprecated skills
- Cross-reference with knowledge/concepts
