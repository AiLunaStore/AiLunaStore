# Knowledge Directory - Semantic Memory Layer

## Purpose
This directory contains your **semantic memory** - structured knowledge, concepts, relationships, and understanding that persists across sessions.

## Structure

### knowledge/concepts/
- Core concepts you understand
- Domain knowledge
- Mental models
- Frameworks and theories

### knowledge/people/
- Information about people you interact with
- Relationships and context
- Preferences and history

### knowledge/projects/
- Long-term project knowledge
- Goals and objectives
- Lessons learned
- Strategic context

### knowledge/systems/
- Understanding of systems you work with
- APIs, tools, and platforms
- Integration patterns
- Best practices

### knowledge/world/
- General world knowledge
- Facts and information
- Context about the world

## Usage Guidelines

1. **Semantic vs Episodic**: Store structured knowledge here, not raw experiences
2. **Cross-referencing**: Link related concepts using markdown links
3. **Versioning**: Update files as understanding evolves
4. **Consolidation**: Periodically review and consolidate knowledge from daily memory files

## Maintenance
- Review weekly during memory consolidation
- Prune outdated information
- Update as understanding deepens
- Ensure consistency across files
