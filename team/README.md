# Team Directory

## Purpose
This directory contains team collaboration artifacts, project materials, meeting notes, and shared resources for the Levin-Luna team.

## Structure

### team/meetings/
- Meeting notes and summaries
- Discussion topics and agendas
- Action items and follow-ups
- Retrospectives and reviews

### team/projects/
- Active project documentation
- Project plans and timelines
- Deliverables and artifacts
- Status reports and updates

### team/decisions/
- Important decisions and rationale
- Decision logs with context
- Alternatives considered
- Outcomes and learnings

### team/resources/
- Shared references and materials
- Templates and checklists
- Guides and how-tos
- External resource links

## Team Collaboration Principles

### 1. Clear Communication
- Document decisions and rationale
- Keep meeting notes actionable
- Update project status regularly
- Flag blockers and dependencies

### 2. Shared Context
- Maintain up-to-date documentation
- Cross-reference related materials
- Provide background when needed
- Ensure both team members have necessary context

### 3. Accountability
- Track action items to completion
- Follow up on commitments
- Report progress transparently
- Acknowledge and learn from mistakes

### 4. Continuous Improvement
- Regularly review processes
- Solicit and incorporate feedback
- Experiment with new approaches
- Document what works and what doesn't

## Meeting Protocol

### Before Meetings
1. Create agenda in `team/meetings/YYYY-MM-DD-topic.md`
2. Share relevant pre-reading materials
3. Define desired outcomes

### During Meetings
1. Take notes in the same file
2. Capture decisions in `team/decisions/`
3. Document action items with owners and deadlines

### After Meetings
1. Distribute notes within 24 hours
2. Follow up on action items
3. Schedule next steps if needed

## Project Management

### Project Initiation
1. Create project folder in `team/projects/project-name/`
2. Define scope, goals, and success criteria
3. Identify stakeholders and dependencies
4. Create initial timeline and milestones

### Project Execution
1. Maintain regular status updates
2. Track progress against milestones
3. Document challenges and solutions
4. Communicate changes to scope or timeline

### Project Closure
1. Document outcomes and learnings
2. Archive project materials
3. Conduct retrospective
4. Celebrate successes

## Decision Making

### Decision Process
1. Document the decision needed in `team/decisions/`
2. Gather relevant information and context
3. Identify alternatives and trade-offs
4. Make decision with rationale
5. Communicate decision to relevant parties
6. Implement and follow up

### Decision Log Format
```markdown
# Decision: [Brief description]

## Date: YYYY-MM-DD
## Decision Makers: [Who was involved]
## Status: [Pending/Approved/Implemented/Rejected]

## Context
[Why this decision is needed]

## Options Considered
1. [Option A] - Pros/Cons
2. [Option B] - Pros/Cons
3. [Option C] - Pros/Cons

## Decision
[Selected option]

## Rationale
[Why this option was chosen]

## Implementation Plan
[Steps to implement, who's responsible, timeline]

## Follow-up
[How success will be measured, review date]
```

## Resource Management

### Resource Types
- **Templates:** Reusable formats for common documents
- **Checklists:** Step-by-step guides for repetitive tasks
- **References:** Important information for quick access
- **Tools:** Software, scripts, and utilities
- **Learning Materials:** Tutorials, courses, and documentation

### Maintenance
- Review resources quarterly for relevance
- Update templates as processes evolve
- Archive deprecated resources
- Add new resources as they're created or discovered
